Переходим в Google Console и создаём новое приложение
Создаём приложение
После создания приложения нам необходимо перейти в API & Services > OAuth consent screen и выбрать External
Нажимаем Create
Задаём имя, почту, лого, почту дублируем в контактной информации
Далее нам необходимо добавить пользователей которые могут авторизоваться в  приложении
Переходим в раздел Credentials > Create credentials > OAuth client ID, необходимо задать тип приложения - WebApplication
Далее нам необходимо задать "http://localhost:8080" для Authorized JavaScript origins и "http://localhost:8080/" для Authorized redirect URLs
Далее в окне с данными для аунтефикации мы скачиваем JSON файл, кладём его в папку с EXE файлом и переименовываем в client_secrets.json

Заполняем config.ini:
log_name = "systemlog.log" <- Это имя нашего лога которое будет нам засорять файловую систему (шутка)
ip = "192.168.100.2" <- Это наш IP адрес который в Windows мы можем узнать командой "ipconfig"
excel_file = "report.xlsx" <- Это название файла нашего отчёта который будет создаваться каждый раз

Заполняем compare.ini:
compare = [
    ["mac", "name"], <- Здесь в поле "mac" мы прописываем mac адрес устройства нашего сотрудника, а в поле "name" - прописываем его имя на которое в xlsx файле будет заменяться mac адрес
    ["mac2", "name2"],
    ["mac3", "name3"],
    ["mac4", "name4"],
    ["mac5", "name5"],
    ["mac6", "name6"]
]
